# EcoFuzz

## AFLFast Seed Scheduling Algorithm
- 프로그램의 실행 경로를 마르코프 체인으로 모델링하고, Low-frequency paths를 탐색하는 시드에 더 많은 에너지를 할당
- 마르코프 체인 모델링
  - 마르코프 체인 : 어떤 상태에서 다른 상태로 넘어갈 때의 확률을 모델링한 수학적 체계
    - 미래의 상태는 오직 현재 상태에만 영향을 받으며, 과거에 어떤 상태들을 거쳐왔는지 상관이 없음
  - 상태 : 퍼저가 발견한 고유한 실행 경로 자체
    - 상태 i는 특정 시드를 넣었을 때 프로그램이 흘러가는 전체 경로를 의미함
  - 전이 : 현재 경로 i를 실행하게 만든 시드 입력값을 변이시켜서 다른 경로 j를 실행하는 새로운 입력값을 만들어내는 행위
  - 전이 확률 : $$p_{ij} = P(X_{n+1} = j \mid X_n = i)$$
    -  $p_{ij}$ : 경로 $i$를 실행하는 시드를 변이시켰을 때, 다음 번 실행에서 경로 $j$를 탈 확률
    -  i와 j가 코드 구조상 밀접해있고, 도달하기 쉬우면 $p_{ij}$ 가 높음
  - 동작 원리
    - 퍼저가 실행되면서 각 경로가 몇 번 실행되었는지 빈도를 세어, 마르코프 체인의 전이 확률을 내부적으로 추정함
    - 전체 실행 중 드물게 도달하는 Low-frequency paths를 찾아냄
    - 전이 확률이 낮은 희귀 경로를 발견한 시드에 에너지를 몰아줌
  - 장점
    - 깊은 코드 경로 도달 : 희귀 경로에 집중하므로 프로그램 깊숙이 숨겨진 취약점을 더 빠르게 발견할 수 있음
    - 퍼징 효율 극대화 : 이미 충분히 테스트된 경로에 불필요한 퍼징 리소스를 낭비하지 않음
  - 한계
    - 유연성 부족 : 퍼징 진행 상황(초기 탐색 vs 중후반 심화 단계)을 유연하게 반영하지 못함
      - 원인 : 실행 빈도라는 단일 기준만으로 에너지를 할당하는 정적인 수학 모델을 사용함
      - 결과 : 복잡한 제약 조건과 실제 가치 있는 길을 구분하지 못해 자원을 낭비함 --> 새로운 경로를 발견하는 데 필요한 평균 에너지 비용이 기하급수적으로 증가
