### 1. `harness/`는 무슨 역할인가? 
- 정의 및 역할
  - `harness/` : 퍼저가 테스트할 수 있도록 입력 -> 함수 호출을 연결해주는 코드
  - 퍼저는 `main` 함수부터 실행하는 프로그램만 테스트 가능함
  - 많은 라이브러리(libpng 등)는 `main`이 없기 때문에 직접 main을 만들어주는 것을 `harness`라고 할 수 있음
- `libpng` 같은 게 main이 없는 이유?
  - libpng는 프로그램이 아니라 라이브러리
  - 독립적으로 실행할 수 없고, 다른 프로그램에서 호출되어야 함
  ```C
  png_read_image(...)
  png_create_read_struct(...)
  ```
  - 위와 같은 함수만 존재하고 main()이 없음
- `harness`의 구조
  ```C
  int main(int argc, char** argv) {
    FILE *fp = fopen(argv[1], "rb");  // 입력 파일
    // 데이터를 읽어서
    // libpng 같은 라이브러리 함수 호출
    png_read_image(...);
  }
  ```
  - 퍼저가 만든 입력 파일을 받고 파일을 읽어서 라이브러리 함수에 전달하여 crash/bug 발생 여부를 확인함

--> 실행 파일만 fuzzing 가능하기 때문에 실행 가능한 프로그램 형태로 만들어야 하고, 이를 **harness** 라고 함

### 2. AFL 퍼저 입력 방식 & 주요 옵션(@@, -f 등)
- AFL 퍼저를 사용하여 타겟 프로그램을 퍼징할 때, 타켓 프로그램이 입력을 받는 방식(File, STDIN)에 따라 AFL의 실행 옵션이 달라져야 함
- AFL 필수 기본 옵션
  - `-i [디렉토리]` : 초기 시드 파일들이 들어있는 입력 디렉토리 경로
  - `-o [디렉토리]` : 퍼징 결과가 저장될 출력 디렉토리 경로
  - `--` : AFL 퍼저의 옵션이 끝났음을 알리는 구분자 (이 뒤에 오는 내용들은 전부 타겟 프로그램을 실행하기 위한 명령어와 인자로 취급됨)  
- 1. 타겟 프로그램의 입력 방식에 따른 3가지 실행 포맷
  - 일반적인 파일 입력 (확장자를 검사하지 않는 경우)
    - 타겟 프로그램이 파일 경로를 인자로 받지만, 파일의 확장자를 검사하지 않을 때 사용함
    - 사용 기호 : `@@`
    - 의미 : `@@`은 AFL이 생성한 변이된 임시 파일의 경로가 들어갈 자리 표시자
    - 예시 
      ```
      ```bash
      ./afl-fuzz -i input_seed -o output -- ./pdftoimage @@
      ```
      - 동작 원리 : AFL은 내부적으로 `.cur_input`과 같은 확장자가 없는 임시 파일을 만든 뒤, 명령어의 `@@` 부분을 해당 파일의 실제 경로(ex: /tmp/.cur_input)로 치환하여 타겟 프로그램을 실행  
- 2. 엄격한 파일 입력 (특정 확장자를 요구하는 경우)
  - `libpng`나 특정 문서 파서처럼, 입력 파일의 확장자가 반드시 특정 포맷이어야만 정상적으로 동작하는 프로그램을 퍼징할 때 사용
  - 사용 옵션 : `-f [고정할 파일명]`
  - 의미 : AFL에게 "네가 만드는 모든 변이 파일의 이름을 내가 지정한 이름(확장자 포함)으로 고정해서 만들어"라고 지시하는 옵션
  - 예시
    ```
    ./afl-fuzz -i input_seed -o output -f input.pdf -- ./pdftoimage -in input.pdf
    ```
    - 동작원리
      - `-f input.pdf`를 통해 AFL은 변이된 데이터를 무조건 `input.pdf`라는 이름으로 생성
      - 타겟 프로그램의 명령어 인자에도 `@@` 대신 우리가 고정한 `input.pdf`를 직접 명시
      - 타겟 프로그램은 확장자가 `.pdf`이므로 정상적인 파일로 인식하고 퍼징
- 3. 표준 입력(STDIN)
  - 프로그램이 파일을 읽는 게 아니라 `scanf`, `cin`, `read(0, ...)` 처럼 키보드나 파이프라인을 통한 표준 입력을 받을 때 사용
  - 사용 기호 : `@@`나 `-f` 옵션 모두 사용하지 않음
  - 예시
    ```
    ./afl-fuzz -i input_seed -o output -- ./cxxfilt
    ```
    - 동작 원리 : 타겟 프로그램 뒤에 아무런 파일 인자를 주지 않으면, AFL은 알아서 변이된 데이터를 타겟 프로그램의 표준 입력(STDIN) 파이프를 통해 직접 밀어 넣음

--> 소규모 실험 돌렸을 때 실패한 이유 : 타겟이 `libpng`(png 확장자를 강하게 검사하는 프로그램)임에도 불구하고, 파일명 고정 옵션 없이 `@@`를 사용하여 퍼징을 시도함   
--> 확장자가 없는 임시 파일을 생성하여 전달했고, libpng는 확장자가 .png가 아니라는 이유로 내부 로직(파싱)을 수행하기도 전에 프로그램을 에러로 종료시켜버림. (결과적으로 유의미한 코드 커버리지를 탐색하지 못함)   
--> 해결 방법 : `-f input.png` 옵션을 주고 타겟 프로그램의 인자로 `input.png`를 명시하여 실행

### 3. 소규모 실험 재실행 
```
./run_fuzzer.sh afl 5m libpng_read_fuzzer png "input.png" "input.png"
```
- `$5` : `"@@"` -> `"input.png"`
- `$6` : `""` -> `"input.png"`
  - `run_afl.sh` 스크립트 내부에서 `INPUT_OPTION="-f input.png"`로 변환되어 AFL 퍼저의 옵션으로 들어감 
- 전과는 다르게 제대로 돌아감
- 결과 해석
  - `cat /output/coverage.txt` : 코드 커버리지 확인
  - `ls -l /output/default/crashes` : 크래시 찾았는지 확인
