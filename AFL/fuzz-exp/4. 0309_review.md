### Q1. `harness/`는 무슨 역할인가? 
- 정의 및 역할
  - `harness/` : 퍼저가 테스트할 수 있도록 입력 -> 함수 호출을 연결해주는 코드
  - 퍼저는 `main` 함수부터 실행하는 프로그램만 테스트 가능함
  - 많은 라이브러리(libpng 등)는 `main`이 없기 때문에 직접 main을 만들어주는 것을 `harness`라고 할 수 있음
- `libpng` 같은 게 main이 없는 이유?
  - libpng는 프로그램이 아니라 라이브러리
  - 독립적으로 실행할 수 없고, 다른 프로그램에서 호출되어야 함
  ```C
  png_read_image(...)
  png_create_read_struct(...)
  ```
  - 위와 같은 함수만 존재하고 main()이 없음
- `harness`의 구조
  ```C
  int main(int argc, char** argv) {
    FILE *fp = fopen(argv[1], "rb");  // 입력 파일
    // 데이터를 읽어서
    // libpng 같은 라이브러리 함수 호출
    png_read_image(...);
  }
  ```
  - 퍼저가 만든 입력 파일을 받고 파일을 읽어서 라이브러리 함수에 전달하여 crash/bug 발생 여부를 확인함

--> 실행 파일만 fuzzing 가능하기 때문에 실행 가능한 프로그램 형태로 만들어야 하고, 이를 **harness** 라고 함
