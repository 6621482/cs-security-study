# add-triage branch

## Commit 1
- `docker-setup/build_with_afl.sh` 에서 `-fsanitize=addreses` 옵션 추가
- 타겟 바이너리 컴파일 시 ASAN 옵션을 추가하면 퍼징 중 발생하는 미세한 Memory Corruption을 즉각적으로 탐지하고 상세한 crash report를 얻을 수 있음
- 과정
  - 1. 새로운 branch 생성 (`add-triage`)
    - 실험 환경의 원본 코드를 보호하고, 수정 내역을 독립적으로 관리하기 위함
  - 2. `docker-setup/build_with_afl.sh`를 수정하여 컴파일러 플래그에 ASAN 옵션을 추가함
    -  **추가 옵션** : `-fsanitize=address`
  - 3. 작업 완료 후 버전을 기록하기 위해 커밋을 남김
    - `git commit -m "Commit #1: docker-setup/build_with_afl.sh 에서 -fsanitize=address 옵션 추가하기"`
  - 4. 도커 이미지 재빌드
    - 수정된 컴파일 스크립트가 타겟 프로그램에 적용되도록 컨테이너와 타켁 바이너리들을 처음부터 다시 컴파일함
    ```
    sudo ./build.sh
    ```
  - 5. 컨네이터 접속
    ```
    sudo ./launch.sh
    ```
  - 6. 확인
    - ASAN이 타겟 바이너리에 정상적으로 Linking 되었는지 확인하기 위해, 컴파일된 `nasm` 실행 파일을 대상으로 심볼 검사를 진행함
    ```
    nm ./nasm/nasm | grep asan
    ```
--> 결과 : ASAN 관련 심볼이 출력됨
