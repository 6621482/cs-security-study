## ASAN(AddressSanitizer)

- c/c++로 제작된 프로그램에서 버그를 디텍션 해주는 툴
- 컴파일러 Clang 을 설치한 뒤 컴파일할 때 `fsanitize=address` 옵션만 붙여주면 됨
- 디텍션할 수 있는 버그
  - Use after free
  - Heap buffer overflow
  - Stack buffer overflow
  - Global buffer overflow
  - Use after return : 이미 반환된 함수의 매개변수를 사용하려 함
  - Use after scope : 다른 지역에서 선언된 변수를 사용하려 함 
  - Initialization order bugs : 객체나 변수의 초기화 순서 문제
  - Memory leaks

  - 퍼징과 함께 쓰는 이유?
    - 소스 코드의 몇 번째 줄에서 어떤 메모리 문제 때문에 죽었는지 분석하기 위해
    - ASAN을 적용해서 빌드한 바이너리에 크래시를 유발하는 입력을 넣으면, 어디서 문제가 발생했는지 아주 상세한 에러 리포트와 스택 트레이스를 출력해줌 

```
#!/bin/bash

mkdir -p /benchmark/bin/asan

export CC=/usr/lib/llvm-18/bin/clang
export CXX=/usr/lib/llvm-18/bin/clang++
export PATH=/usr/lib/llvm-18/bin:$PATH

export CFLAGS="-fsanitize=address -g -O1"
export CXXFLAGS="-fsanitize=address -g -O1"
export LDFLAGS="-fsanitize=address"

./build_libtiff.sh || exit 1
cp /build_output/* /benchmark/bin/asan/ || exit 1

./build_binutils.sh || exit 1
cp /build_output/* /benchmark/bin/asan/ || exit 1

./build_libxml2.sh || exit 1
cp /build_output/* /benchmark/bin/asan/ || exit 1

./build_libpng.sh || exit 1
cp /build_output/* /benchmark/bin/asan/ || exit 1

./build_libming.sh || exit 1
cp /build_output/* /benchmark/bin/asan/ || exit 1

./build_cyclonedds.sh || exit 1
cp /build_output/* /benchmark/bin/asan/ || exit 1

./build_tcpdump.sh || exit 1
cp /build_output/* /benchmark/bin/asan/ || exit 1

./build_nasm.sh || exit 1
cp /build_output/* /benchmark/bin/asan/ || exit 1

./build_libncurses.sh || exit 1
cp /build_output/* /benchmark/bin/asan/ || exit 1
```
- `export CC=/usr/lib/llvm-18/bin/clang`
  - `CC` : C언어 컴파일러 지정 
- `export CXX=/usr/lib/llvm-18/bin/clang++`
  - `CXX` : C++ 언어 컴파일러 지정 (`clang++` 사용) 
- `PATH` : 터미널에서 명령어를 쳤을 때 실행 파일을 찾는 경로
- `export CFLAGS="-fsanitize=address -g -O1"
  - `CFLAGS` : C를 컴파일할 때 컴파일러에게 넘겨주는 옵션
  - `-fsanitize=address` : 메모리를 읽고 쓰는 모든 어셈블리어 코드 사이사이에 메모리 오염을 검사하는 코드를 끼워넣음
  - `-g` : 디버그 심볼 (프로그램이 크래시가 났을 때 "메모리 0x4012ab에서 에러 발생"이라고 나오는 대신, "main.c 파일의 42번째 줄에서 에러 발생"이라고 소스 코드와 매핑해서 보여주기 위함)
  - `-O1` : 최적화 레벨
- `export CXXFLAGS="-fsanitize=address -g -O1"`
  -  `CFLAGS` : C++을  컴파일할 때 컴파일러에게 넘겨주는 옵션
- `export LDFLAGS="-fsanitize=address"`
  - `LDFLAGS` : 컴파일이 끝난 뒤 최종 실행 파일을 만드는 링크 단계에서 링커(Linker)에게 넘겨주는 옵션
