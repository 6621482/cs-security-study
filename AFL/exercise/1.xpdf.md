## Fuzzing exercise_1

### 1. 실습 목적
AFL++ 를 이용해서 PDF 처리 프로그램 `pdftotext`를 퍼징
- `pdftotext` : PDF 파일을 읽어서 텍스트로 변환하는 프로그램

### 2. 개념
- Target program : 퍼징의 대상 프로그램 (실습에서는 `pdftotext`)
- seed input : 퍼징을 시작할 때 사용하는 초기 입력 파일
- mutation : seed input을 기반으로 입력 데이터를 변형하여 새로운 테스트 입력을 생성하는 과정
- instrumentation : AFL 컴파일러를 이용해 타겟 프로그램을 다시 빌드하여 실행 경로(coverage)를 기록하는 코드 삽입
- Coverage-guided fuzzing : 프로그램 실행 중 어떤 코드 경로가 실행되었는지를 기반으로 새로운 입력을 선택하는 퍼징 방식

### 3. 실습 전체 흐름
1. 타겟 프로그램(Xpdf) 다운로드
2. 빌드 환경 준비
3. Xpdf를 먼저 정상 빌드해서 동작 확인
4. seed PDF 파일 준비
5. AFL++ 설치
6. Xpdf를 AFL 컴파일러로 다시 빌드
7. afl-fuzz 실행
8. 퍼징 결과(output, queue, crashes, hangs) 확인

### 4. 핵심 내용 
- target을 AFL용으로 다시 빌드
  - 퍼징에서 프로그램 실행 경로를 추적하기 위해 타겟 프로그램을 AFL 컴파일러로 다시 빌드해야 함 
  ```
  export CC=afl-clang-fast
  export CXX=afl-clang-fast++
  ```
  - `export` : 환경 변수를 설정하고 이후 실행되는 프로그램에도 적용되도록 하는 명령어
  - 이 상태에서 configure, make를 실행하면 AFL이 coverage 추적 코드를 프로그램에 삽입함 
- input directory (-i 옵션)
  ```
  -i pdf_examples
  ```
  - 퍼징을 시작할 때 사용하는 seed input 디렉터리
  - AFL은 이 디렉터리의 파일을 기반으로 mutation을 수행
- output directory (-o 옵션)
  ```
  -o output
  ```
  - 퍼징 결과를 저장하는 디렉터리
  ```
  output/
   └ default/
       ├ queue/
       ├ crashes/
       └ hangs/
   ```
  - queue : 새로운 코드 경로를 발견한 입력
  - crashes : 프로그램 crash를 발생시킨 입력
  - hangs : 프로그램이 일정 시간 이상 멈춘 입력

### 5. 결과 해석 
- total execs : 지금까지 프로그램이 실행된 총 횟수
- exec/sec : 초당 프로그램 실행 횟수 (퍼징 속도를 나타냄)
- corpus count : AFL이 저장하고 있는 입력 데이터 개수
- saved crashes : 발견된 crash 개수
- saved hangs : 프로그램이 멈춘 입력 개수
