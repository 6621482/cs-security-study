# Database
- **Database** : 컴퓨터 시스템에 저장된 정보나 데이터의 집합

## Database Management System (DBMS) 
- **DBMS(Database Management System)** : 데이터베이스를 효율적으로 관리하기 위해 사용되는 소프트웨어
- 역할
  - 데이터 관리 : 새로운 정보를 기록하고 기존 내용을 수정하거나 삭제할 수 있음 
  - 동시 접근 지원 : 여러 사용자가 동시에 데이터베이스에 접근하여 데이터를 사용할 수 있게 함
  - 복잡한 데이터 조회 : 웹 서비스의 검색 기능처럼 복잡한 요구사항을 충족하는 데이터를 정확하게 찾아낼 수 있음
- 관계형(Relational) vs 비관계형(Non-Relational)
  - 관계형 : 행과 열의 집합인 테이블(Table) 형식
    - 정해진 구조에 따라 정형화된 데이터 저장에 유리함
    - 대표적인 DBMS : MySQL, MariaDB, PostgreSQL, SQLite
  - 비관계형 : 테이블 형식이 아닌 키-값(Key-Value) 등 다양한 형태
    - 고정되지 않은 구조나 대량의 분산 데이터 처리에 유리
    - 대표적인 DBMS : MongoDB, CouchDB, Redis

### Relational DBMS (RDBMS)
- RDBMS : 데이터를 표(Table) 형식으로 저장하고 관리하는 데이터베이스 모델
- 조건
  - 테이블 구조 : 데이터가 행(Row)과 열(Column)의 집합인 테이블 형태로 관리되어야 함
  - SQL 지원 : 테이블 형식의 데이터를 조작할 수 있는 관계 연산자인 **SQL(Structured Query Language)** 을 제공해야 함
- 데이터 관리 방식
  - 데이터가 하나의 큰 표에 모두 담기는 것이 아니라, 주제에 따라 여러 테이블로 나누어 저장됨
  - 고유 키(Primary Key) 활용: 각 테이블은 고유한 식별자를 가짐
  - 테이블 간 연결: 서로 다른 테이블이라도 공통된 고유 키를 기준으로 서로 연관될 수 있음

## SQLite
### 테이블 생성하기
- 테이블 : 데이터를 저장하는 기본 단위로, 행과 열로 구성되어 있음
- 각 행에는 하나의 데이터 레코드가, 각 열에는 데이터의 속성이 저장됨
- **테이블 생성 명령어 - `CREATE TABLE`**
  ```
  CREATE TABLE 테이블_이름 (
    열_이름 데이터_유형 제약조건,
    열_이름 데이터_유형 제약조건,
    ...
  );
  ```
  - 테이블_이름 : 생성할 테이블의 이름으로, 데이터베이스 내에서 고유해야 함
    - ex. `student`, `products`
  - 열_이름 : 테이블에서 데이터를 저장할 속성(필드) 이름
    - ex. `id`, `name`, `age`
  - 데이터_유형 : 해당 열에 저장될 데이터의 유형을 지정함
    - ex. `INTEGER` 정수형 데이터 / `TEXT` 문자열 데이터 / `REAL` 실수형 데이터 / `DATE` 날짜 데이터
  - 제약조건 : 데이터의 유효성을 검사하는 규칙을 지정함
    - `PRIMARY KEY` : 각 행을 고유하게 식별하는 역할을 하도록 만듦 (고유 키)
      - 고유 키 : 테이블의 특정 행을 고유하게 식별할 수 있는 필드로 설정하는 것 (고유 키로 설정된 피드는 중복된 값 저장 불가능)  
    - `NOT NULL` : 값이 반드시 입력되어야 하도록 만듦
    - `UNIQUE` : 값이 중복되면 안되도록 만듦
    - `DEFAULT` : 기본값을 지정함
- **테이블 추가 명령어 - `INSERT INTO`**
  - 테이블에 새로운 데이터를 삽입하는 SQL 명령어
  - 테이블에 정의된 각 열에 따라 데이터를 추가할 수 있음 
  ```
  INSERT INTO 테이블_이름 (열1, 열2, 열3, ...) VALUES (값1, 값2, 값3, ...);
  ```
  - `테이블_이름` : 데이터를 삽입할 테이블의 이름
  - `열1, 열2, 열3, ...` : 데이터를 삽입할 열의 이름들
  - `값1, 값2, 값3, ...` : 삽입할 데이터의 값 (각 열의 데이터 유형과 일치해야 함)
- **데이터 조회 명령어 - `SELECT`**
  - 저장된 정보를 원하는 대로 필터링해서 조회할 수 있음
  ```
  SELECT 열_이름1, 열_이름2, 열_이름3, ... FROM 테이블_이름;
  ```
  - `열_이름` : 테이블에서 데이터를 조회할 때 어떤 열을 조회할 것인지 지정함
  - `테이블_이름` : 어느 테이블에서 데이터를 조회할 것인지 지정함
  - ex. `SELECT * FROM students;` -> `student` 테이블에 저장된 모든 데이터를 가져온다는 의미 
    - `*` : 모든 열을 가져오라는 의미
  - ex. `SELECT * FROM students WHERE id = 2;` -> `student` 테이블에 저장된 데이터에서 `id`가 2인 데이터만 가져온다는 의미 
    - `WHERE` : 데이터를 가져올 때 필터링을 적용함
- **데이터 수정 명령어 - `UPDATE`**
  ```
  UPDATE 테이블_이름 SET 열_이름1 = 새로운값, 열_이름2 = 새로운값 WHERE 조건;
  ```
  - `UPDATE 테이블_이름` : 데이터를 수정할 테이블의 이름을 지정함
  - `SET 열_이름1 = 새로운값` : 변경할 열과 새로운 값을 지정함 (여러 열 동시 수정 가능)
  - `WHERE 조건` : 데이터를 수정할 행을 조건으로 지정함
  - ex. `UPDATE students SET name = 'Mark' WHERE id = 3;` -> `student` 테이블에 저장된 데이터에서 `id`가 3인 데이터의 `name`을 `Mark`로 수정하겠다는 의미
- **데이터 삭제 명령어 - `DELETE`**
  ```
  DELETE FROM 테이블_이름 WHERE 조건;
  ```
  - `DELETE FROM 테이블_이름` : 데이터를 삭제할 테이블의 이름을 지정함
  - `WHERE 조건` : 데이터를 수정할 행은 조건으로 지정함
  - ex. `DELETE FROM students WHERE id = 3;` -> `student` 테이블에 저장된 데이터 중 `id`가 3인 데이터를 삭제하겠다는 의미
