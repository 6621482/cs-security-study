# Web

## World Wide Web
- 정의 : 인터넷을 기반으로 구현된 서비스 중 HTTP를 이용하여 정보를 공유하는 서비스
- 웹 서버(Web Server) : 정보를 제공하는 주체
  - Nginx : 사용자의 요청을 가장 먼저 받는 웹 서버 소프트웨어
  - Gunicorn & Flask: 실제 비즈니스 로직(연산, 데이터 처리)을 수행하는 WAS(Web Application Server) 영역
  - DBMS (Database): 유저 정보나 게시글 같은 데이터를 저장하고 관리하는 창고 역할  
- 웹 클라이언트(Web Client) : 정보를 받는 이용자
- HTTP : 웹상에서 서로 통신하기 위해 정해둔 일종의 규칙

### 웹의 발전 과정 
- 초기 : 저장된 문서의 내용을 단순히 출력하여 이용자에게 보여줌
- 현재 : 다양한 분야로 확장되었으며, 사용자가 직접 정보를 검색하고 제품을 구매하는 등 능동적인 상호작용이 가능해짐
- 변화: 오프라인에서의 많은 상호작용이 현재는 디지털 형태의 웹 서비스로 구현되고 있음

### 웹 보안의 중요성 
- 민감 정보 처리 : 물건 구매 시 주소, 카드 번호와 같은 개인정보가 웹을 통해 서버로 전달
- 피해 발생 가능성 : 정보가 안전하게 보호되지 않을 경우 사용자에게 심각한 피해를 줄 수 있음
- 목적: 정보 교환 과정에서 민감한 정보가 유출되거나 악용되지 않도록 보호하는 것

### 웹 서비스
- 프론트엔드(Front-end) : 이용자의 요청을 받는 부분
  - 웹 리소스(Web resource) : 웹에 갖춰진 정보 자산
    - 웹 브라우저 주소창에 특정 주소를 입력하는 것은 해당 서버에 존재하는 특정 경로의 리소스를 가져오겠다는 요청과 동일함
    - 식별 방법 : 모든 웹 리소스는 URI(Uniform Resource Identifier)**라는 고유한 식별자를 가지고, 이를 통해 구분됨
    - 구성 : 웹 페이지에 표시되는 글자, 색상, 이미지, 동영상 등 화면을 구성하는 모든 데이터가 리소스에 해당함
    - 예시
      - Hyper Text Markup Language(HTML) : 태그와 속성을 사용하여 구조화된 문서를 만듦 (문서의 뼈대와 구조)
      - Cascading Style Sheets(CSS) : 글자 색상, 배경, 이미지 크기 및 위치 등을 설정하여 브라우저가 이를 참고해 화면을 시각화함 (디자인&시각화)
      - JavaScript(JS) : 사용자의 클릭이나 데이터 입력에 어떻게 반응할지 정의함 (동작 구현)
      - 그 외 : 문서, 이미지, 동영상, 폰트 등 
- 백엔드(Back-end) : 이용자의 요청을 처리하는 부분

### 웹 클라이언트와 서버의 통신
1. 접속 (클라이언트) : 이용자가 브라우저(Chrome, Edge 등)를 사용하여 웹 서버에 접속함
2. 리소스 요청 (클라이언트) : 브라우저는 이용자의 요청을 해석하여 HTTP 형식으로 웹 서버에 리소스를 보내달라고 요청함
3. 요청 해석 (서버) : 서버는 HTTP를 통해 전달된 이용자의 요청 내용을 분석함
4. 동작 수행 (서버) : 해석된 결과에 따라 적절한 동작을 수행 (내부 연산 처리 등)
5. 리소스 전달 (서버) : 처리가 완료되면 이용자에게 전달할 결과를 다시 HTTP 형식으로 클라이언트에게 보냄
6. 시각화 (클라이언트) : 브라우저는 서버로부터 받은 HTML, CSS, JS 등의 웹 리소스를 해석하고 시각화하여 이용자에게 최종 화면을 보여줌
