# Use After Free

## Use After Free
- **Use After Free** : 동적으로 할당된 메모리를 해제한 후에도 해당 메모리를 가리키는 포인터를 통해 다시 접근할 때 발생하는 문제
- **Dangling Pointer** : 유효하지 않은 메모리 영역을 가리키는 포인터
  - 할당 : malloc 함수 등을 통해 메모리를 동적으로 할당받으면, 해당 메모리의 주소가 포인터에 저장됨
  - 해제 : free 함수를 호출하여 메모리를 반환함
  - 문제 : free 함수는 chunk를 반환하지만, 그 주소를 담고 있던 포인터 변수를 초기화하지는 않음
  - 해제 후에도 포인터는 이미 반환된 메모리 주소를 그대로 가짐
