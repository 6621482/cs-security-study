# Linux Shell

## Shell
### 운영체제의 level
- 유저 레벨
  - 사용자가 조작할 수 있는 부분
  - 일반 응용 프로그램이 실행되는 공간
  - 권한 : 제한적, 하드웨어를 직접 건드릴 수 없음
  - 안전성 : 여기서 프로그램 하나가 오류로 꺼져도, 컴퓨터 전체는 멈추지 않고 해당 프로그램만 종
- 커널 레벨
  - 컴퓨터의 모든 자원(CPU, 메모리, 하드디스크 등)에 접근할 수 있는 최고 관리자 모드
  - 권한 : 무제한, 시스템의 모든 명령어 실행 가능, 모든 하드웨어 제어 가능
  - 역할
    - 프로세스 관리
    - 메모리 관리
    - 하드웨어 장치 제어
  - 위험성 : 여기서 에러가 발생하면 컴퓨터 전체가 멈춤

### Shell
- **Shell** : 커널과 유저 사이를 연결해주는 프로그램
- 종류
  - **GUI(Graphical User Interface Shell)** - 그래픽 기반
    - 장점 : 사용자들이 직관적으로 기능을 파악하고 사용할 수 있음
    - 단점 : 그래픽 작업이 CPU, GPU를 상대적으로 많이 소모하여 실행속도가 느림
  - **CLI(Command-Line Interface Shell)** - 텍스트 기반
    - ex. sh, bash, zsh 
    - 장점 : 실행속도가 빠르고 정밀한 작업을 수행하기 편리함
    - 단점 : 비직관적
- **터미널(Terminal)**
  - CLI 셸을 GUI 셸 위에서 사용할 수 있도록 하는 프로그램
