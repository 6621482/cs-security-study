## Dreamhack - Netcat

### 문제 유형
- 시스템 해킹 기초
- 원격 서비스 통신
- TCP 소켓 기반 입력 / 출력

### 문제 요약
원격 서버에서 실행 중인 프로그램에 접속하여 특정 문자열을 입력하면 플래그를 반환
서버 주소와 포트가 주어지며, `nc`를 이용해 해당 서비스에 연결하는 것이 핵심

### 코드 분석 포인트
- 서버 측 프로그램은 이미 실행 중
- 사용자의 입력을 받아 문자열 비교를 수행
- 특정 문자열이 입력되면 플래그를 출력

### 접근 과정
1. 문제 설명을 통해 Host와 Port 정보 확인
2. `nc` 명령을 사용해 원격 서버에 TCP 연결
3. 서버가 입력을 대기 중임을 확인
4. 요구된 문자열을 정확히 입력
5. 서버로부터 플래그 출력 확인

### 어려웠던 점
- 처음 웹사이트에 바로 접속했을 때 페이지 접속이 안돼서 당황스러웠음
- 로컬에서 실행하는 게 아니라 원격 서버에 접속하는 문제라는 것을 깨달음

### 배운 점
- 시스템 해킹 문제에서는 프로그램이 서버에서 이미 실행 중인 경우가 많음
- 사용자는 프로그램을 실행하는 것이 아니라 입력을 전달하는 역할만 수행함
- 단순한 문자열 입력 문제라도 네트워크 통신 개념이 기본이 됨

### 쓰인 개념
- netcat (`nc`)
- TCP 통신
- 포트 번호
- 표준 입력 / 출력

### 추가 내용
**`nc`란 무엇인가?**
- `netcat(nc)` : TCP 또는 UDP 연결을 생성하여 네트워크를 통해 데이터를 송수신할 수 있는 도구
- 특정 프로토콜에 종속되지 않고, 단순히 소켓 연결을 제공하는 **범용 네트워크 유틸리티**
- 시스템 해킹 문제에서는 원격 서버에서 실행 중인 프로그램에 **입력을 전달하고 출력을 확인하기 위한 도구**로 자주 사용
**Q. `nc host port`는 무엇을 하는 명령인가?**
- 지정한 호스트와 포트 번호로 TCP 연결을 생성
- 사용자가 입력한 내용을 그대로 서버에 전달
- 서버의 출력 결과를 터미널에 그대로 출력
