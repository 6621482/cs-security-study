## Dreamhack - Baby Linux

### 문제 유형
- 리눅스 기초
- 웹 기반 명령 실행
- 문자열 필터링 우회

### 문제 요약
사용자 입력을 받아 리눅스 명령어를 실행하는 웹 서비스가 주어짐
`app.py`가 제공되며, `flag.txt` 파일을 출력하는 것이 목표

### 코드 분석 포인트
- 사용자 입력은 `echo $(입력값)` 형태로 쉘에서 실행됨
- 실행 문자열에 특정 문자열 `flag`가 포함되면 결과를 출력하지 않고 차단
- 이는 단순한 문자열 포함 여부만 검사하는 필터링

### 접근과정
1. 웹서비스에서 실행되는 명령의 형태 파악
2. `app.py`를 확인하여 어떤 조건에서 출력이 차단되는지 분석
3. `hint.txt`를 통해 flag 파일의 위치 정보를 확인
4. 단순한 파일 출력 방식은 문자열 필터링에 걸린다는 점 확인
5. 필터링 방식의 한계를 이용해 우회 가능한 방법을 고민

### 어려웠던 점
- flag 파일의 위치는 알 수 있었지만, 직접적인 접근은 차단되었기에 이를 해결하기 위한 방법을 탐색함

### 배운 점
- 문자열 기반 필터링은 구조적으로 우회 가능성이 존재함
- 리눅스 명령어의 다양한 표현 방식(와일드카드 등)은 보안 문제로 이어질 수 있음

### 쓰인 개념
- ls, cat, echo 등 기초적인 명령어
- 와일드카드
- 상대 경로

### 추가 내용
**Q. echo 뒤의 `$()`는 왜 사용되는가?**
- `$(명령어)` : 쉘에서 명령 치환을 의미
  - 괄호 안의 명령어를 먼저 실행한 뒤, 그 출력 결과를 문자열로 치환
- 예시 :
  - `echo ls` : 문자열 `ls`를 그대로 출력
  - `echo $(ls)` : `ls`를 실행한 결과를 `echo`로 출력
    - (1) `ls` 실행
    - (2) `ls`의 출력 결과를 가져옴
    - (3) 그 결과를 `echo`의 인자로 전달
    - (4) `echo`가 출력
